Libra is the "methodology development" library we actively develop in the group. It is intended for various types of calculations in quantum and classical chemical dynamics and beyond. Learn more about this code on its website (follow the link).

Specific for the group at UB/CCR:

  1. Group version of the core is located at: 
    /projects/alexeyak/Software/libra-code


PYXAID is the second tool for nonadiabatic quantum dynamics (NAQD) in atomistic systems we develop in the group. The code website (follow the link) contains several detailed step-by-step tutorials. The tutorials cover basic and advanced topics related to practical utilization of the code. The tutorials also cover some basics of calculations with the QE code.

Specific for the group at UB/CCR:

  1. Group version of the core is located at: 
    /projects/alexeyak/Software/pyxaid-code
  2. When compiling your own copy, modify the Makefile to set include/library paths for boost and Python as: 
    # UB CCR
p1=/util/academic/boost/v1.57.0/include/boost
pl1=/util/academic/boost/v1.57.0/lib

# UB CCR
p2=/util/academic/python/2.7.6/include/python2.7
pl2=/util/academic/python/2.7.6/lib
  3. Before running the code, load the Boost libraries 
    module load boost
  4. Before running the code from an arbitrary location, set up PYTHONPATH environment variable. In the ".bash_profile" file (in your home directory) add the following line: 
    export PYTHONPATH=$PYTHONPATH:/projects/alexeyak/Software/pyxaid-code
    The previous operation, can also be added to the ".bash_profile" file, if you don't want to do this every time you use PYXAID. Reload your terminal, for the changes to take an effect.


For electronic structure calculations, we we utilize the open-source Quantum Espresso (QE) code. It is based on the plane-wave basis and density functional theory (DFT). Many theoretical and practical topics are nicely explained in the tutorials available on the official QE website, but you can also find some examples and discussion on the PYXAID website


CCR computing

  1. Start your SLURM (submit) scripts with the following header: 
    #!/bin/sh
#SBATCH --partition=valhalla
#SBATCH --clusters=chemistry
#SBATCH --time=20:00:00
#SBATCH --nodes=1
#SBATCH --ntasks-per-node=1
#SBATCH --mem=1500
#SBATCH --mail-user=???@buffalo.edu

echo "SLURM_JOBID="$SLURM_JOBID
echo "SLURM_JOB_NODELIST="$SLURM_JOB_NODELIST
echo "SLURM_NNODES="$SLURM_NNODES
echo "SLURMTMPDIR="$SLURMTMPDIR
echo "working directory="$SLURM_SUBMIT_DIR

NPROCS=`srun --nodes=${SLURM_NNODES} bash -c 'hostname' |wc -l`
echo NPROCS=$NPROCS

    This will allow your computations to be run on the group's dedicated nodes ("valhalla" partition)

    Replace ??? with your UB ID, so that ???@buffalo.edu is your valid email address

    Adjust all other parameters of the job
