# 3.4. Libra code on UB CCR [Libra code](https://github.com/Quantum-Dynamics-Hub/libra-code) is a modular methodology prototyping library for quantum dynamics, written in C++ and Python. Libra enables quantumâ€“classical dynamics calculations with model and atomistic Hamiltonians. Libra boasts a comprehensive set of trajectory surface hopping methods and algorithms for nonadiabatic dynamics. ## A. Using Libra To install Libra, we use the instruction in [here](https://github.com/Quantum-Dynamics-Hub/libra-code). It is already installed on CCR and in order to load it, one needs to follow these steps: ``` module use /projects/academic/cyberwksp21/MODULES module load libra/v5.5.0 # Other versions are also available: v5.3.0 and the devel branch of the code in github repository. ``` Using the above command it will load the `libra` environment and one can test the loading by: ``` python -c "from liblibra_core import *" ``` ## B. Using on Jupyter notebook In order to use it on Jupyter notebook, you need to follow these steps: You first need to install Jupyter notebook manually on your terminal. This is a one time setup and you don't need to do this anymore you want to use Jupyter on CCR. ``` module use /projects/academic/cyberwksp21/MODULES module load libra pip install jupyterlab ``` Then, create your own kernel by: ``` python -m ipykernel install --user --name=libra_kernel ``` Since the Jupyter lab on CCR resets every module before running the Jupyter notebook interface, you need to properly specify the `PYTHONPATH` and `LD_LIBRARY_PATH` variables to the `_build/src` directory for each version of libra installed in your `~/.bashrc` file as follows (e.g. v5.5.0): ``` export PYTHONPATH=/projects/academic/cyberwksp21/SOFTWARE_NEW_ENV/libra/v5.5.0/_build/src:$PYTHONPATH export LD_LIBRARY_PATH=/projects/academic/cyberwksp21/SOFTWARE_NEW_ENV/libra/v5.5.0/_build/src:$LD_LIBRARY_PATH ``` Then, from OnDemand, load a JupyterLab job and after jupyter is run, select the `libra_kernel` whenever you want to run a calculations that requires Libra's modules. **Note:** This only works with Jupyter Lab (legacy) version for now and will not be working by June 2024. ### C. Update from 12/04/2024 (based on Mohammad's notes) To use the new version (5.8.1 and later) of Libra, with the updated CMakeLists.txt file on CCR, load the module: ``` module load libra/devel2 ``` There is no conda environment loaded and it uses the CCR precompiled modules except for boost (with version 1.80.0). The old one uses the conda environment using: ``` module load libra/devel ``` To use the new Libra build in Jupyter notebook, make sure that you correctly give the paths in your `~/.bashrc` file (just adding a 3 to `_build` suffices), e.g.: ``` export PYTHONPATH=/projects/academic/cyberwksp21/SOFTWARE_NEW_ENV/libra/devel/_build3/src:$PYTHONPATH export LD_LIBRARY_PATH=/projects/academic/cyberwksp21/SOFTWARE_NEW_ENV/libra/devel/_build3/src:$LD_LIBRARY_PATH ``` **Just for those who want to compile their own fork of the code** No changes to CMakeLists.txt is required. Just need to do this: #### 1. Load the module and cd to the `_build` folder ``` module load libra/devel2 # To load all the dependencies and libraries and boost cd /path/to/your/libra-code/and/then/to/_build ``` #### 2. Make sure the _build directory path is empty #### 3. Then build as usual: ``` cmake ../ # No worries about the warnings, that's the best I could deliver today and the compilation is unaffected and the liblibra_core is imported make -j8 ``` After the compilation is done, make sure you have correctly recompiled it by doing this: ``` module purge cd /path/to/your/libra-code/and/then/to/_build/src python -c "from liblibra_core import *" ```
